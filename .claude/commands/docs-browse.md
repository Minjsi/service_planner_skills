# docs-browse: 기획 문서 탐색

이 skill은 프로젝트 코드를 기반으로 기획 문서를 탐색하고 조회합니다.

**사용법:**
- `/docs-browse` — 프로젝트 코드를 대화형으로 질문
- `/docs-browse PRJ-001` — 프로젝트 코드를 인자로 직접 전달

인자는 `$ARGUMENTS` 변수에서 읽습니다.

## 실행 절차

아래 절차를 순서대로 수행하세요.

### 1단계: 설정 확인

`.claude/docs-config.md` 파일을 읽어 `repo_url`과 `clone_path` 값을 확인합니다.

- **값이 비어있거나 파일이 없으면:** 아래 메시지를 출력하고 **즉시 중단**합니다.
  > 기획 문서 repository가 설정되지 않았습니다. 먼저 `/docs-setup`을 실행하여 설정해주세요.

### 2단계: Clone 경로 확인 및 Git Pull (최신화)

`clone_path` 디렉토리가 존재하는지 확인합니다.

- **디렉토리가 없으면:** 아래 메시지를 출력하고 **즉시 중단**합니다.
  > clone 경로(`{clone_path}`)가 존재하지 않습니다. `/docs-setup`을 다시 실행하여 설정해주세요.

`clone_path` 경로에서 `git pull`을 실행합니다.

- **성공 시:** 다음 단계로 진행
- **실패 시:** 아래 경고를 표시하되 로컬 데이터로 계속 진행합니다.
  > ⚠ git pull에 실패했습니다. 로컬에 저장된 버전으로 진행합니다.

### 3단계: 프로젝트 코드 확인

- `$ARGUMENTS`에 프로젝트 코드가 전달되었으면 해당 값을 사용합니다.
- `$ARGUMENTS`가 없으면 사용자에게 프로젝트 코드를 물어봅니다:
  > 조회할 프로젝트 코드를 입력해주세요. (예: `PRJ-001`)

### 4단계: 문서 목록 조회

`clone_path` 하위에서 입력받은 프로젝트 코드에 해당하는 폴더를 찾고, 그 안의 `.md` 파일을 **재귀적으로** 탐색합니다. 파일 경로는 프로젝트 코드 폴더 기준 상대 경로로 표시합니다.

분기 처리:

**폴더가 없는 경우:**
> `{프로젝트코드}` 폴더를 찾을 수 없습니다. 프로젝트 코드를 확인해주세요.

**`.md` 파일이 0개인 경우:**
> `{프로젝트코드}` 프로젝트에 문서(.md)가 없습니다.

**`.md` 파일이 1개인 경우:**
해당 문서의 전체 내용을 바로 읽어서 표시합니다.

**`.md` 파일이 여러 개인 경우:**
목록을 보여주고 어떤 문서를 볼지 선택을 요청합니다:
> `{프로젝트코드}` 프로젝트에 다음 문서들이 있습니다:
> 1. 기획서.md
> 2. 요구사항.md
> 3. phase2/설계문서.md
>
> 어떤 문서를 보시겠습니까? 번호 또는 파일명을 입력해주세요.

### 5단계: 문서 출력

선택된 문서의 전체 내용을 읽어서 출력합니다.
