# docs: 기획 문서 통합 관리

기획 문서 repository 설정, 탐색, Q&A를 하나의 명령어로 수행합니다.

**사용법:**
- `/docs setup` — repository 설정
- `/docs browse [프로젝트코드]` — 문서 탐색
- `/docs ask [프로젝트코드] [문서명]` — 특정 문서 Q&A
- `/docs` — 사용 가능한 명령어 안내

인자는 `$ARGUMENTS` 변수에서 읽습니다. 첫 번째 단어가 서브커맨드입니다.

---

## 서브커맨드 분기

`$ARGUMENTS`의 첫 번째 단어를 확인하여 아래 섹션 중 하나를 실행합니다.

- `setup` → **A. Setup** 섹션 실행
- `browse` → **B. Browse** 섹션 실행
- `ask` → **C. Ask** 섹션 실행
- 비어있거나 그 외 → **D. 도움말** 섹션 실행

---

## A. Setup

### A-1: 기존 설정 확인

`.claude/docs-config.md` 파일을 읽어 현재 설정 상태를 확인합니다.
- `repo_url`과 `clone_path`가 이미 설정되어 있으면 현재 설정을 보여주고, 변경할지 물어봅니다.
- 값이 비어있으면 새로 설정을 진행합니다.

### A-2: 로컬 경로 확인

먼저 사용자에게 이미 clone된 로컬 경로가 있는지 물어봅니다.

> 기획 문서 repository가 이미 로컬에 clone 되어 있나요?
> - **있다면**: clone된 경로를 입력해주세요. (예: `/{workspace}/project-docs`)
> - **없다면**: `없음`이라고 입력해주세요.

**이미 clone 되어있는 경우:**
- 입력받은 경로가 git repository인지 확인 (`.git` 폴더 존재 여부).
  - git repository가 맞으면 → `git remote -v`로 repo URL을 자동으로 읽어옴 → **A-4로 이동**
  - git repository가 아니면 → "해당 경로는 git repository가 아닙니다." 안내 후 재입력 유도

**clone 되어있지 않은 경우 → A-3으로 진행**

### A-3: 새로 Clone

> 기획 문서가 저장된 git repository URL을 입력해주세요.
> 예: `https://github.com/org/project-docs.git`

> 로컬에 clone할 경로를 입력해주세요.
> 기본값: `~/project-docs`

`git clone <repo_url> <clone_path>` 실행.
- 실패 시 에러 메시지를 표시하고 URL과 경로를 다시 확인하도록 안내

### A-4: Git Pull (최신화)

clone 경로에서 `git pull`을 실행하여 최신 상태로 업데이트합니다.

### A-5: Repository 구조 확인

clone된 경로의 최상위 폴더 목록을 표시합니다.

### A-6: 설정 저장

`.claude/docs-config.md` 파일을 아래 형식으로 저장합니다:

```markdown
---
repo_url: <입력받은 URL>
clone_path: <입력받은 경로>
---
```

> 설정이 완료되었습니다. `/docs browse` 또는 `/docs ask`로 문서를 조회할 수 있습니다.

## B. Browse

### B-1: 설정 확인

`.claude/docs-config.md` 파일을 읽어 `repo_url`과 `clone_path` 값을 확인합니다.

- **값이 비어있거나 파일이 없으면:**
  > 기획 문서 repository가 설정되지 않았습니다. 먼저 `/docs setup`을 실행하여 설정해주세요.

  즉시 중단합니다.

### B-2: Clone 경로 확인 및 Git Pull (최신화)

`clone_path` 디렉토리가 존재하는지 확인합니다.

- **디렉토리가 없으면:** 아래 메시지를 출력하고 **즉시 중단**합니다.
  > clone 경로(`{clone_path}`)가 존재하지 않습니다. `/docs setup`을 다시 실행하여 설정해주세요.

`clone_path` 경로에서 `git pull`을 실행합니다.

- **성공 시:** 다음 단계로 진행
- **실패 시:** 아래 경고를 표시하되 로컬 데이터로 계속 진행합니다.
  > ⚠ git pull에 실패했습니다. 로컬에 저장된 버전으로 진행합니다.

### B-3: 프로젝트 코드 확인

`$ARGUMENTS`에서 `browse` 다음 단어를 프로젝트 코드로 사용합니다.

- 없으면 사용자에게 물어봅니다:
  > 조회할 프로젝트 코드를 입력해주세요. (예: `PRJ-001`)

### B-4: 문서 목록 조회

프로젝트 코드 폴더 하위의 `.md` 파일을 **재귀적으로** 탐색합니다.

**폴더가 없는 경우:**
> `{프로젝트코드}` 폴더를 찾을 수 없습니다. 프로젝트 코드를 확인해주세요.

**`.md` 파일이 0개인 경우:**
> `{프로젝트코드}` 프로젝트에 문서(.md)가 없습니다.

**`.md` 파일이 1개인 경우:**
해당 문서의 전체 내용을 바로 읽어서 표시합니다.

**`.md` 파일이 여러 개인 경우:**
> `{프로젝트코드}` 프로젝트에 다음 문서들이 있습니다:
> 1. 기획서.md
> 2. 요구사항.md
> 3. phase2/설계문서.md
>
> 어떤 문서를 보시겠습니까? 번호 또는 파일명을 입력해주세요.

### B-5: 문서 출력

선택된 문서의 전체 내용을 읽어서 출력합니다.

## C. Ask

### C-1: 설정 확인

`.claude/docs-config.md` 파일을 읽어 `repo_url`과 `clone_path` 값을 확인합니다.

- **값이 비어있거나 파일이 없으면:**
  > 기획 문서 repository가 설정되지 않았습니다. 먼저 `/docs setup`을 실행하여 설정해주세요.

  즉시 중단합니다.

### C-2: Clone 경로 확인 및 Git Pull (최신화)

`clone_path` 디렉토리가 존재하는지 확인합니다.

- **디렉토리가 없으면:** 아래 메시지를 출력하고 **즉시 중단**합니다.
  > clone 경로(`{clone_path}`)가 존재하지 않습니다. `/docs setup`을 다시 실행하여 설정해주세요.

`clone_path` 경로에서 `git pull`을 실행합니다.

- **성공 시:** 다음 단계로 진행
- **실패 시:** 아래 경고를 표시하되 로컬 데이터로 계속 진행합니다.
  > ⚠ git pull에 실패했습니다. 로컬에 저장된 버전으로 진행합니다.

### C-3: 프로젝트 코드 확인

`$ARGUMENTS`에서 `ask` 다음 단어를 프로젝트 코드로 사용합니다.

- 없으면 사용자에게 물어봅니다:
  > 조회할 프로젝트 코드를 입력해주세요. (예: `PRJ-001`)

### C-4: 문서 선택

프로젝트 코드 폴더 하위의 `.md` 파일을 **재귀적으로** 탐색합니다.

**폴더가 없는 경우:**
> `{프로젝트코드}` 폴더를 찾을 수 없습니다. 프로젝트 코드를 확인해주세요.

즉시 중단합니다.

**`$ARGUMENTS`에 문서명이 포함된 경우:**
`ask` 다음의 두 번째 인자 이후를 문서명으로 사용합니다. 해당 파일이 존재하는지 확인하고, 없으면 아래 에러를 표시합니다:
> `{문서명}` 파일을 찾을 수 없습니다. 파일명을 확인해주세요.

**`$ARGUMENTS`에 문서명이 없는 경우:**
- `.md` 파일이 0개 → "해당 프로젝트에 문서가 없습니다" 안내 후 중단
- `.md` 파일이 1개 → 자동 선택
- `.md` 파일이 여러 개 → 목록을 보여주고 선택 요청

### C-5: 문서 로드 및 Q&A 시작

선택된 문서의 전체 내용을 읽어 컨텍스트에 로드합니다.

> 📄 `{문서명}` 문서를 로드했습니다.
> 이 문서에 대해 궁금한 점을 자유롭게 질문해주세요.

이후 사용자의 질문에 대해 **로드된 문서 내용을 기반으로** 답변합니다. 문서에 없는 내용에 대한 질문에는 "해당 내용은 문서에 포함되어 있지 않습니다"라고 안내합니다.

## D. 도움말

인자가 없거나 인식되지 않는 서브커맨드인 경우 아래를 표시합니다:

> **기획 문서 관리 명령어:**
>
> | 명령어 | 설명 |
> |--------|------|
> | `/docs setup` | 기획 문서 repository 설정 |
> | `/docs browse [프로젝트코드]` | 프로젝트 코드 기반 문서 탐색 |
> | `/docs ask [프로젝트코드] [문서명]` | 특정 문서 지정 Q&A |
>
> 또는 개별 명령어를 사용할 수 있습니다: `/docs-setup`, `/docs-browse`, `/docs-ask`
