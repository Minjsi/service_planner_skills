# docs-ask: 기획 문서 Q&A

이 skill은 특정 프로젝트의 기획 문서를 로드하여 자연어로 질의응답을 수행합니다.

**사용법:**
- `/docs-ask` — 프로젝트 코드, 문서를 순차적으로 질문
- `/docs-ask PRJ-001` — 프로젝트 코드 전달 → 문서 선택 후 Q&A
- `/docs-ask PRJ-001 기획서.md` — 프로젝트 코드 + 문서명 직접 전달

인자는 `$ARGUMENTS` 변수에서 읽습니다.

## 실행 절차

아래 절차를 순서대로 수행하세요.

### 1단계: 설정 확인

`.claude/docs-config.md` 파일을 읽어 `repo_url`과 `clone_path` 값을 확인합니다.

- **값이 비어있거나 파일이 없으면:** 아래 메시지를 출력하고 **즉시 중단**합니다.
  > 기획 문서 repository가 설정되지 않았습니다. 먼저 `/docs-setup`을 실행하여 설정해주세요.

### 2단계: Clone 경로 확인 및 Git Pull (최신화)

`clone_path` 디렉토리가 존재하는지 확인합니다.

- **디렉토리가 없으면:** 아래 메시지를 출력하고 **즉시 중단**합니다.
  > clone 경로(`{clone_path}`)가 존재하지 않습니다. `/docs-setup`을 다시 실행하여 설정해주세요.

`clone_path` 경로에서 `git pull`을 실행합니다.

- **성공 시:** 다음 단계로 진행
- **실패 시:** 아래 경고를 표시하되 로컬 데이터로 계속 진행합니다.
  > ⚠ git pull에 실패했습니다. 로컬에 저장된 버전으로 진행합니다.

### 3단계: 프로젝트 코드 확인

`$ARGUMENTS`를 파싱하여 첫 번째 값을 프로젝트 코드로 사용합니다.

- 프로젝트 코드가 없으면 사용자에게 물어봅니다:
  > 조회할 프로젝트 코드를 입력해주세요. (예: `PRJ-001`)

### 4단계: 문서 선택

`clone_path` 하위에서 프로젝트 코드 폴더를 찾고, `.md` 파일을 **재귀적으로** 탐색합니다.

**폴더가 없는 경우:**
> `{프로젝트코드}` 폴더를 찾을 수 없습니다. 프로젝트 코드를 확인해주세요.

그리고 즉시 중단합니다.

**`$ARGUMENTS`에 문서명이 포함된 경우:**
두 번째 인자 이후를 문서명으로 사용합니다. 해당 파일이 존재하는지 확인하고, 없으면 아래 에러를 표시합니다:
> `{문서명}` 파일을 찾을 수 없습니다. 파일명을 확인해주세요.

**`$ARGUMENTS`에 문서명이 없는 경우:**
- `.md` 파일이 0개 → "해당 프로젝트에 문서가 없습니다" 안내 후 중단
- `.md` 파일이 1개 → 자동 선택
- `.md` 파일이 여러 개 → 목록을 보여주고 선택 요청:
  > 다음 문서들이 있습니다:
  > 1. 기획서.md
  > 2. 요구사항.md
  > 3. phase2/설계문서.md
  >
  > 어떤 문서에 대해 질문하시겠습니까?

### 5단계: 문서 로드 및 Q&A 시작

선택된 문서의 전체 내용을 읽어 컨텍스트에 로드합니다.

그 후 아래와 같이 안내합니다:

> 📄 `{문서명}` 문서를 로드했습니다.
> 이 문서에 대해 궁금한 점을 자유롭게 질문해주세요.

이후 사용자의 질문에 대해 **로드된 문서 내용을 기반으로** 답변합니다. 문서에 없는 내용에 대한 질문에는 "해당 내용은 문서에 포함되어 있지 않습니다"라고 안내합니다.

사용자가 다른 주제로 전환하거나 대화를 종료할 때까지 Q&A를 계속합니다.
